# Data Card

## Datasets Referenced

### Million Song Dataset (MSD)
- **Source**: http://millionsongdataset.com/
- **Content**: Audio features and metadata for ~1M songs
- **License**: Research / non-commercial use only
- **Status in this repo**: NOT included. Stubs in `src/ingestion/msd_loader.py`
- **Field mapping**:
  - `song_id` → `track_id`
  - `artist_name`, `artist_id` → `artists.name`, `artists.artist_id`
  - `title` → `tracks.title`
  - `tempo`, `energy`, `loudness`, `year`, `duration` → `tracks.*`

### Taste Profile Subset
- **Source**: http://millionsongdataset.com/tasteprofile/
- **Content**: 48M user-song play counts from Echo Nest
- **License**: Academic research only — do not redistribute
- **Status**: NOT included. Stub in `src/ingestion/taste_profile_loader.py`

### MusicBrainz
- **Source**: https://musicbrainz.org/
- **License**: CC0 1.0 (fully open, safe for any use)
- **Content**: Artist/recording/release metadata, MBID identifiers
- **Status**: NOT included. Stub in `src/ingestion/musicbrainz_loader.py`

### ListenBrainz
- **Source**: https://listenbrainz.org/
- **License**: CC0 / user-consented listen data
- **Content**: User listen history
- **Status**: NOT included. Stub in `src/ingestion/listenbainz_loader.py`

## Synthetic Sample Data

The sample data in `data/sample/` is **fully synthetic**, generated by `src/ingestion/sample_generator.py` using:
- Faker (names, usernames)
- NumPy random distributions matching real-world statistics:
  - Play counts: Zipf(1.5) power law
  - Audio features: Beta distributions matching Spotify/MSD ranges
  - Genre distribution: Weighted by real streaming market share
  - Listen patterns: Popularity-weighted user-track sampling

This data is safe to commit, share, and use without any licensing concerns.

## Schema Alignment

Our DuckDB schema is designed to accept data from all four datasets above. The ingestion adapters (when implemented) must map their native schemas to our canonical tables: `artists`, `tracks`, `users`, `listens`.
